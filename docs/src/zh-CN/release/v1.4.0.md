v1.4.0 重点围绕 plugin 扩展能力、CLI 输出体验和代码生成稳定性进行增强。

## 新增特性

### cmd/jzero

* 新增外部 plugin 支持。`jzero` 现在可以从 `PATH` 发现并执行 plugin 命令，并提供 `cmd/jzero/pkg/plugin` 作为 plugin 开发复用包 [#441](https://github.com/jzero-io/jzero/pull/441)

![](https://oss.jaronnie.com/image-20260422150933066.png)

* 重构 CLI 输出，新增 logo、结构化进度与错误展示、debug 配置展示，并在执行前打印命令，便于排障 [#451](https://github.com/jzero-io/jzero/pull/451)

* 重构 `gen` 流程并优化生成输出，`--git-change` 增加专门的 `git-change` 模式，便于审查实际生成变更 [#403](https://github.com/jzero-io/jzero/pull/403)

![](https://oss.jaronnie.com/image-20260422151018001.png)

## 问题修复

### cmd/jzero

* 修复项目存在 `types.go` 时的 API 生成问题，并支持多个 API 文件共享同一个 `go_package`

* 修复 `model.go.tpl` 并调整 model 模板，使生成代码更贴合当前的 model 生成流程 [#438](https://github.com/jzero-io/jzero/pull/438) [#439](https://github.com/jzero-io/jzero/pull/439)

* 修复生成流程中的工作目录处理和清理边界问题，例如移除多余文件时 `etc` 目录为空的场景 [#447](https://github.com/jzero-io/jzero/pull/447)

## 文档更新

* 新增 CLI Tool Guide，说明如何基于 jzero template 构建 CLI 工具

* 新增 AI base project 快速开始与 Vercel serverless function 相关博客内容，并补充 overview、AI capabilities 等说明

* 增加 templates market 入口，并更新 README、model 指南等相关文档

## 更新步骤

```shell
jzero upgrade --channel v1.4.0
jzero gen
```
